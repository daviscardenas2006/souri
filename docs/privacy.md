# SOURI Privacy

SOURI is being built around a privacy-first, local-first product philosophy.

This document explains the public privacy stance of the product at its current stage.

## Privacy-first by design

SOURI is designed to help reduce the amount of sensitive information that reaches AI systems in the first place.

The main product goal is preventive: detect relevant data at the input layer, provide visible user control, and allow action before submission.

## Local-first direction

SOURI is being developed with a local-first approach.

That means the product is intended to operate as close as possible to the user environment, reducing unnecessary external exposure and keeping privacy actions near the point where the data appears.

## What SOURI does

At a public product level, SOURI is designed to:

- inspect supported input surfaces for sensitive data patterns
- help users sanitize content before submission
- show visible feedback when relevant data is detected
- maintain an audit trail of relevant privacy events
- allow users to create reusable rules from real events

## What this public repository includes

This repository currently focuses on:

- product direction
- public documentation
- interface previews
- screenshots
- architecture and privacy notes

## What remains private for now

At this stage, the core implementation remains private.

That includes internal product logic, implementation details, and other early-stage technical components that are not yet part of the public product surface.

## Product boundaries

SOURI is intended to improve practical privacy control in AI workflows, but it should not be interpreted as a guarantee of perfect protection in all cases.

Like any privacy-oriented tool, its effectiveness depends on product scope, user behavior, supported surfaces, and the nature of the data being handled.

## Public commitment

The public direction of SOURI is centered on:

- reducing unnecessary exposure
- giving users visible control
- building around local-first behavior
- making privacy actions practical and reusable
- staying clear about the current product boundary
