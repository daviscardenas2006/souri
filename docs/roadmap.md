# SOURI Roadmap

SOURI is in an early but real product stage.

This public roadmap summarizes the current direction of development without exposing internal planning details that are better kept private during the early phase.

## Current foundation

The current product foundation already includes:

- sensitive data detection on active inputs
- prompt sanitization actions
- visible in-context interaction through a toast UI
- a local audit viewer for privacy events
- rule creation from real events
- editable rule and preset surfaces

## Public presence

The current public milestone is focused on establishing SOURI as a credible product presence.

That includes:

- public GitHub repository
- product-facing README
- public architecture, privacy, and roadmap notes
- curated screenshots
- a cleaner external narrative for future presentation

## Near-term priorities

The next product priorities are expected to focus on improving workflow quality rather than expanding public marketing too early.

That includes areas such as:

- stronger product polish
- more consistent UI surfaces
- better rule management flows
- improved audit usability
- clearer product states and edge-case handling

## Product direction

SOURI is being shaped around a practical privacy workflow:

1. detect sensitive information
2. allow action before submission
3. record meaningful events
4. turn repeated actions into reusable rules
5. improve user control over time

## Broader direction

Over time, the public direction of SOURI may expand into:

- broader support across interaction surfaces
- more refined privacy controls
- stronger configuration flows
- improved product maturity for broader release
- a more complete external product presence

## Roadmap philosophy

SOURI is being built with a product-first mindset.

That means the priority is not to add surface-level complexity too early, but to strengthen the core workflow, preserve credibility, and grow the public presence in a controlled way.
